# Track: Launch & Socialization

Get Verifrog in front of early users across hardware engineering, open-source EDA, and F# communities.

## Deliverables

1. **Launch blog post** — "Why we built Verifrog": the problem with testbenches, the approach (F# + Verilator), the debug tools (checkpoint/fork/sweep), quick getting-started. Canonical reference linked from all channels.

2. **Demo assets** — 30-second GIF and/or 2-minute terminal walkthrough showing the core workflow.

3. **Channel-specific posts** — Tailored for r/FPGA, r/fsharp, Hacker News, LinkedIn, FOSSi, F# Weekly, Verilator Discussions. Not copy-paste — each audience gets the angle that matters to them.

4. **Comparison table** — Honest positioning vs UVM, cocotb, and plain Verilog testbenches.

## Success criteria

- GitHub stars and early issues/feedback from people outside our circle
- At least one community discussion with substantive engagement
- Identify the first external user who tries it on their own design
